Map

map是Go语言提供的一种抽象数据类型，它表示一组无序的键值对。map对value的类型没有限制，但是对key的类型有严格要求：key的类型应该严格定义了作为“ == ”和“ != ”两个操作符的操作数时的行为，因此函数、map、切片不能作为map的key类型。

Go Map 的底层机制

Go运行时使用一张哈希表来实现抽象的map类型。运行时实现了map操作的所有功能，包括查找、插入、删除、遍历等。在编译阶段，Go编译器会将语法层面的map操作重写成运行时对应的函数调用。这里先介绍老版本的map实现：

核心结构 - hmap

桶结构 - bmap

  • 每个桶通常可以存储 8 个键值对
  • 当桶已满时，会通过overfolw指针式连接新的溢出桶

工作机制

a) 哈希与定位：

可以看到，使用算出的 hash 的低位区的值用于选择桶，高位区的值用于确定桶中 key 的位置

b)解决哈希冲突：
  • 链地址法：每个桶是一个链表头，溢出桶通过指针链接
  • 开放寻址：在桶内线性探测寻找空位
c)扩容机制：
  • 增量扩容：当count > 6.5 * 2^B （负载因子过高）时触发,此时运行时会建立一个两倍于现有规模的bucket数组，但真正的排空和迁移工作是在进行assign 和 delete 操作时逐步进行的。原 bucket 数组会挂在 hmap 的 oldbuckets 指针下面，直到原 buckets 数组中所有数据都迁移到新数组，原 bucktes 数组才会被释放。
  • 等量扩容：当溢出桶太多时，重新排列减少溢出桶
  • 扩容时渐近式的，每次操作旧桶时迁移1-2个桶到新桶

并发Map的实现方案

除了简单的全局锁（sync.RWMutex），基于桶的并发Map主要有以下几种高级实现方式：

1.分片锁（Sharded Locking）

将一个大的 Map分成多个小 Map（分片），每个分片有自己的锁

  • 读写操作限制在单个分片上竞争，大幅减少冲突
  • 实现相对简单，性能接近无锁结构

2.读写锁 + 拷贝写 （Copy-On-Write）

读操作完全无锁，写操作通过原子替换整个 map 实现

  • 适用于读多写少，且写操作不复杂的场景
  • 适用于 Map 尺寸不大的场景（否则拷贝的开销将不可承受）

最后，Go标准库已经提供了 sync.Map ，来支持 Map 的并发读写；此外，也有第三方库 github.com/orcaman/concurrent-map 提供了分片 Map 的实现。
